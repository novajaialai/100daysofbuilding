# Meeting Recorder

Press **Super+R** to start recording a meeting; press **Super+R** again to stop
and save. Sibling to the `~/voice` dictation setup (Super+Z), same
faster-whisper engine.

## What it does

- Records **both sides** of the meeting: your mic + system audio (whatever
  comes out of your speakers/headphones), mixed to one mono 16 kHz track.
- Live-transcribes in ~20 s chunks while you're still in the meeting.
- On stop, appends a `## Summary` section to the transcript (what it was
  about, key points, decisions, action items) — generated by the `claude`
  CLI, falling back to local ollama (`llama3.2:3b`) if offline. If both fail
  you get a critical notification and a loud note in the transcript; the
  transcript itself is always complete first.
- Transcript and audio land in `~/Meetings/<date_time>/`;
  `~/Meetings/latest` always points at the current/most recent session
  (`tail -f ~/Meetings/latest/transcript.md` works mid-meeting).
- A small window shows a blinking red ● REC dot, elapsed time, live status,
  and the transcript as it grows. Closing the window = stop & save.
- After saving, the window stays open (green dot, "Saved ✓") so you can keep
  reading, and the transcript .md auto-opens in GNOME Text Editor. Close the
  window whenever you're done — pressing Super+R again meanwhile starts a
  fresh recording.

## Fails loudly — three layers

1. **Internal watchdog** (every 2 s): recorder process alive, audio file
   actually growing, transcriber not stuck. Failure → window turns red,
   critical notification, alarm sound on repeat. Pure digital silence for
   3 chunks (mic muted?) → orange warning + alarm, auto-clears if audio returns.
2. **External watchdog** (`watchdog.sh`): if the supervisor process itself
   dies, you get an alarm sound, a critical notification, AND a zenity error
   dialog; the orphaned recorder is killed so it can't record unsupervised.
3. **Startup check** (`meeting.sh`): if recording doesn't actually start
   within 20 s of the keypress, same loud alarm. No silent failure modes:
   if you see the blinking dot and hear no alarm, it is recording.

## Files

- `meeting.sh` — Super+R toggle (GNOME custom keybinding `custom2`)
- `supervisor.py` — GTK4 window + ffmpeg recorder + faster-whisper transcriber
  + internal watchdog
- `watchdog.sh` — external process watchdog
- `meeting.log` — supervisor stderr (check here after an alarm)
- per-session `recorder.log` — ffmpeg errors

## Tuning (env vars, set in meeting.sh or globally)

- `MEETING_MODEL` — whisper model (default `base`; `small` = better accuracy,
  slower)
- `MEETING_CHUNK_SECS` — chunk length / transcription latency (default 20)
- `MEETING_SILENCE_CHUNKS` — consecutive silent chunks before the
  "hearing nothing" alarm (default 3; 0 disables)

## Notes

- Words can be clipped at chunk boundaries; the full wav chunks are kept in
  the session dir if a passage needs re-transcribing with a bigger model.
- The session folder keeps all audio; delete `chunks/` after a meeting if you
  only want the transcript.
