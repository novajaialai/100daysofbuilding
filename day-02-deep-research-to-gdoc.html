05 — Decisions & craft

§How it took shape

  • Clarify before researching"Enterprise AI security tools" was too broad. One question surfaced the real target — a 911 provider — and the whole report re-aimed around public-safety threats.
  • Verify, don't just generateEvery load-bearing claim faced a 3-vote adversarial panel. 2 of 25 were refuted and cut. The proposal cites primary CISA/NENA/NIST sources, not vibes.
  • Honest about the gapThe specific vendor names weren't independently benchmarked in this pass — so they're framed as "a shortlist to evaluate," which is the correct posture for a proposal anyway.
  • Reused the existing Google authNo new OAuth dance — the local token already had documents + drive scopes. Verified it, refreshed it, used it.
  • Built a reusable publisherWrote md_to_gdoc.py once. Next "put this on Google Docs" request is now a one-liner.
06 — The technical unlock

§markdown → styled Google Doc, the right way

The Docs API addresses everything by character index. Insert-and-style-as-you-go drifts every later index and silently misaligns your bold and headings. The fix: insert ALL the text first, then apply styling by absolute index — recorded during a single parse pass.

~/google/md_to_gdoc.py — the crux
# walk markdown once; record absolute ranges as you build the text
line_start = idx
clean, spans = parse_inline_bold(content)
plain.append(clean + '\n')
for (s, e) in spans:
  bold_reqs.append((line_start + s, line_start + e))
idx += len(clean) + 1  # advance past text + newline
# then: one insertText, one batchUpdate of styles, share via Drive
drive.permissions().create(type='anyone', role='reader')

Captured as a durable learning via /compound → ~/.claude/learnings/design-patterns/, so the next occurrence is a lookup, not a re-derivation.
